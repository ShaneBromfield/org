1 Install postgresql

1.1 Install postgresql and one tool for administration 

sudo apt-get install postgresql pgadmin3

2 Setup the server to work locally

2.1 Setup the password for the database 

sudo su postgres -c psql template1

which means

2.1.1 sudo

super user do

2.1.2 su postgres

connect as the user postgres which is the user that runs the postgresql service

2.1.3 -c

Execute the command

2.1.4 psql template1

psql is a cli command to connect to a database template1. Template1 is a template database created in the process of installation of postgresql

2.1.5 Sum up

To sum up, we will connect to the database template1 as the linux user postgres…

2.2 Now we can change the password

Here is the result: 

tony@linux-mi178(1.01,) 15:00:09 ~/bin (master) $ sudo su postgres -c psql template1
could not change directory to "/home/tony/Dropbox/repositories/git/slash-my-home-slash-bin"
psql (8.4.8)
Type "help" for help.
postgres=#

Now type 

ALTER USER postgres WITH PASSWORD '$PASSWORD;

Remember the $PASSWORD value, you will need it for the user

2.3 Setup the password for the user postgres

Delete the passwd of the user 

sudo passwd -d postgres

Setup a new one and type the same passwd as in the previous step ($PASSWORD). 

sudo su -c passwd -c postgres

3 Create a user and a database

Postgresql comes with some simple scripts to help you in creating users and databases

3.1 Create a user 

sudo -u postgres createuser -D -A -P $NEW_USER

-D will be allowed to create databases -A do not know yet -P to prompt for typing a password

3.2 Create a database 

sudo -u postgres createdb -O $NEW_USER $NEW_DATABASE

3.3 More simply

Connect as postgres 

sudo su - postgres

Then 

createuser -D -A -P $NEW_USER
createdb -O $NEW_USER $NEW_DATABASE

4 Drop a database and a user

Beware of the order. You won't be able to drop a database if a user is already logged in.

4.1 Log as postgres

Connect as postgres 

sudo su - postgres

4.2 Drop a database 

dropdb $NEW_DATABASE

4.3 Drop a user 

dropuser $NEW_USER
